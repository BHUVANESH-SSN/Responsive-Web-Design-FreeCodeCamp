About the Project

The JPMorgan Chase Midas Core backend project simulates transaction handling in a banking environment.

It includes asynchronous processing, validation, and an external API for applying incentives.

// Simulated explanation of transaction streaming
Tech Stack
  • Java 17
  • Spring Boot
  • Apache Kafka
  • JUnit
  • H2 Database with JPA
  • Maven

These technologies helped simulate real-world backend systems effectively.

Kafka Integration

KafkaListener consumes incoming transactions asynchronously.

Used @KafkaListener annotation for topic-based message consumption.

@KafkaListener(topics = "transactions") public void receive(Transaction txn) {...}
Validation and Database

Validated sender/receiver details before storing data.

Used H2 database and JPA repositories for data persistence.

Optional<User> user = repo.findById(id); if (user.getBalance() >= txn.amount)
Incentive API

Integrated external REST API provided via a JAR file.

Used RESTTemplate to send request and update balance accordingly.

restTemplate.postForObject(url, request, Response.class)
REST Endpoint

Created an endpoint to query user balance using userId.

Mapped to `/balance?userId=` and handled via Spring Controller.

@GetMapping("/balance") public double getBalance(@RequestParam String userId)
